More Hot Takes

Your React app doesn't need 2MB of JavaScript to display a blog post.

A blog post is literally just text. HTML was invented for this exact purpose.

CSS Grid and Flexbox are already in your browser.

You don't need Bootstrap, Tailwind, or any other CSS framework. Learn CSS once and stop downloading the same layout patterns over and over.

Build tools are a scam.

Webpack, Vite, Rollup... just serving files was already solved in 1991. Your HTML file doesn't need to be "built".

npm install anxiety

Every time you run npm install, you're trusting 1,000+ packages written by strangers. That's not engineering, that's faith.

The Principles

  • HTML First: Start with semantic HTML. It's accessible by default.
  • CSS Second: Style with pure CSS. It's more powerful than you remember.
  • JavaScript Last: Only add JS when HTML and CSS genuinely can't do it.
  • Size Matters: Every kilobyte costs your users time and money. Respect both.
  • Fast by Default: A static HTML file loads instantly. Complexity is a choice.
  • Dependencies are Liabilities: Each dependency is a point of failure, a security risk, and maintenance burden.
  • View Source: Your code should be readable without a sourcemap. If you can't view-source it, you don't really own it.
  • Works Forever: HTML from 1995 still renders. Your React app will need a rewrite in 3 years.
  • Mobile First: Half your users are on phones. Make it work on a 3G connection.
  • Brutalist Honesty: Expose the structure. HTML is beautiful without the makeup.
